# 상세 요구사항 (Requirements)

본 문서는 '한국 영화 정보 플랫폼' 과제의 상세 기능 및 기술 요구사항을 정의합니다.

## 1. 필수 기능 (Must Have)

### 1.1 공통
- **반응형 디자인**: 모바일(\~768px)과 데스크톱(768px\~) 환경을 모두 지원해야 합니다.
- **로딩 상태 표시**: 데이터 페칭 중에는 스피너(Spinner) 또는 스켈레톤(Skeleton) UI를 통해 사용자에게 진행 상태를 알려야 합니다.
- **에러 상태 처리**: API 호출 실패 등 예외 상황에 대해 적절한 에러 처리와 사용자 친화적인 안내 메시지를 표시해야 합니다.
- **네비게이션 바**: 모든 페이지에서 상단 또는 측면 네비게이션을 통해 각 메뉴(메인, 검색, 즐겨찾기)로 접근이 가능해야 합니다.

### 1.2 박스오피스 페이지 (메인 페이지, 경로: `/`)
- **탭 전환**: 일별, 주간, 주말 박스오피스 순위를 탭 형태로 전환하여 확인할 수 있어야 합니다.
- **날짜 선택 (Date Picker)**: 사용자가 원하는 특정 날짜의 박스오피스 데이터를 조회할 수 있어야 합니다.
- **순위 목록 표시**:
  - 표시 항목: 순위, 영화명, 개봉일, 당일 관객수, 누적 관객수
  - 순위 변동 표시: 전일 대비 순위의 상승, 하락, 신규 진입, 유지 상태를 직관적으로 표시해야 합니다.
- **상세 이동**: 영화 아이템 클릭 시 해당 영화의 상세 페이지로 이동합니다.
- **즐겨찾기**: 목록에서 즉시 즐겨찾기에 추가하거나 제거할 수 있는 버튼이 있어야 합니다.

### 1.3 영화 검색 페이지 (경로: `/search`)
- **검색 기능**: 영화명(국문/영문)을 입력하여 검색할 수 있어야 합니다.
- **필터링 옵션**:
  - 개봉년도 (연도 선택 가능)
  - 영화 유형 (장편, 단편, 옴니버스 등)
  - 제작 국가
- **결과 목록**: 검색된 영화들의 목록을 표시하며, 페이지네이션(Pagination) 또는 무한 스크롤(Infinite Scroll) 중 하나를 선택하여 구현합니다.
- **예외 처리**: 검색 결과가 없을 경우 사용자에게 이를 알리는 메시지를 표시합니다.

### 1.4 영화 상세 페이지 (경로: `/movie/:movieCd`)
- **영화 기본 정보**:
  - 영화명 (한글/영문)
  - 개봉일, 상영시간, 장르, 관람등급
- **제작 정보**:
  - 감독 정보
  - 주요 출연진 (배우 명단)
  - 제작사 정보
- **즐겨찾기**: 해당 영화를 즐겨찾기에 추가하거나 제거할 수 있는 기능을 제공합니다.
- **뒤로가기**: 이전 페이지로 돌아갈 수 있는 버튼을 제공합니다.

### 1.5 즐겨찾기 페이지 (경로: `/favorites`)
- **목록 표시**: 사용자가 즐겨찾기(관심 영화)로 등록한 영화들의 목록을 보여줍니다.
- **데이터 영속성**: `LocalStorage`를 활용하여 브라우저를 새로고침하거나 껐다 켜도 데이터가 유지되어야 합니다.
- **관리 기능**: 즐겨찾기 목록에서 특정 영화를 제거할 수 있어야 합니다.
- **빈 상태 처리**: 즐겨찾기한 영화가 없을 경우 안내 메시지를 표시합니다.
- **상세 이동**: 영화 아이템 클릭 시 상세 페이지로 이동합니다.

## 2. 선택 기능 (Nice to Have) - 가산점
- **영화인 검색**: 감독이나 배우 이름으로 검색하고 상세 정보를 확인할 수 있는 기능
- **영화 비교**: 두 개 이상의 영화를 선택하여 정보를 나란히 비교해볼 수 있는 기능
- **통계 차트**: 박스오피스 순위 추이나 관객수 변화를 `Chart.js`, `Recharts` 등을 사용하여 시각화
- **최근 본 영화**: 사용자가 최근에 조회한 영화 목록을 히스토리 형태로 제공
- **테스트 코드**: `Jest` 및 `React Testing Library`를 활용한 단위 테스트 또는 통합 테스트 작성
- **기타**: 사용자 경험을 향상시킬 수 있는 창의적인 추가 기능

## 3. 화면별 상세 스펙

### 3.1 박스오피스 카드
| 필드 | 설명 | API 필드 |
| :--- | :--- | :--- |
| **순위** | 현재 박스오피스 순위 | `rank` |
| **순위 변동** | 전일 대비 순위 증감분 | `rankInten` |
| **영화명** | 영화의 한글 제목 | `movieNm` |
| **개봉일** | 영화의 국내 개봉 날짜 | `openDt` |
| **당일 관객수** | 해당 날짜의 관객 수 | `audiCnt` |
| **누적 관객수** | 개봉 이후 현재까지의 총 관객 수 | `audiAcc` |

### 3.2 영화 상세
| 필드 | 설명 | API 필드 |
| :--- | :--- | :--- |
| **영화명** | 영화의 한글 제목 | `movieNm` |
| **영문명** | 영화의 영문 제목 | `movieNmEn` |
| **개봉일** | 영화의 개봉 날짜 | `openDt` |
| **상영시간** | 영화의 총 상영 시간 (분) | `showTm` |
| **장르** | 영화의 장르 분류 | `genres` |
| **감독** | 영화의 감독 정보 | `directors` |
| **배우** | 주요 출연 배우 명단 | `actors` |
| **관람등급** | 영화 관람 가능 연령 등급 | `audits` |

## 4. 상태 관리 요구사항
- 전역 상태 관리는 **Jotai**를 사용하여 구현합니다.
- **관리 대상 상태**:
  - 즐겨찾기(Favorites) 목록 데이터
  - 현재 선택된 박스오피스 날짜
  - 박스오피스 타입 (일별 / 주간 / 주말)
  - (선택) 검색 필터 및 검색어 상태
